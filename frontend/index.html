Come usare questo laboratorio

Ogni sezione rappresenta un algoritmo crittografico con indicatori di sicurezza. Sperimenta liberamente per capire:

  • Algoritmi storici (NON SICURO): Cesare, Vigenère, Hash MD5/SHA-1 - vulnerabili agli attacchi moderni
  • Algoritmi moderni (SICURO): AES-256, RSA-2048+ - standard industriali attuali
  • Simulazioni attacchi: Brute force, Dictionary attack - per comprendere le vulnerabilità

                Algoritmi Moderni (Sicuri)
            


            
            

                

                    

                        

                            

                                AES-256 File Encryption
                            

                            
Advanced Encryption Standard - Cifratura simmetrica

                        

                        SICURO
                    

                

                

                    

                        
Perché È sicuro

                        
    
                            
  • 256 bit di chiave - 2^256 combinazioni possibili (impraticabile per brute force)
    • 
                            
  • Standard governativo - approvato da NSA per documenti TOP SECRET
    • 
                            
  • Uso attuale - WhatsApp, BitLocker, file system cifrati, VPN
    • 
                            
  • Best practice - usa password lunghe (16+ caratteri) con lettere, numeri e simboli
    • 
                        

                    


                    

                        

                            
Cifratura File

                            

                                

                                    
                                    
                                

                                

                                    
                                    
                                    Consiglio: usa lettere maiuscole, minuscole, numeri e simboli

                                RSA
                            

                            
Crittografia asimmetrica a chiave pubblica

                        

                        SICURO (2048+ bit)
                    

                

                

                    

                        
Perché È sicuro

                        
    
                            
  • Chiave asimmetrica - chiave pubblica per cifrare, privata per decifrare
    • 
                            
  • 2048 bit minimo - standard attuale (4096 bit per massima sicurezza)
    • 
                            
  • Uso attuale - HTTPS/TLS, firma digitale, scambio chiavi SSH, PGP/GPG
    • 
                            
  • Limitazione - cifra solo dati piccoli; per file grandi, usa RSA + AES (ibrido)
    • 
                            
  • Nota quantistica - vulnerabile ai computer quantistici futuri (post-quantum crypto in sviluppo)
    •

Best Practices per la Sicurezza

                        

                        

                            

                                

                                    
Cosa FARE

                                    
    
                                        
  • Usa AES-256 per cifrare file/dati grandi
    • 
                                        
  • Usa RSA-2048+ per scambio chiavi e firma digitale
    • 
                                        
  • Usa SHA-256+ per hash e integrità
    • 
                                        
  • Password lunghe (16+ caratteri) con mix di caratteri
    • 
                                        
  • Aggiorna regolarmente algoritmi e dimensioni chiavi
    • 
                                        
  • Combina RSA + AES per cifratura ibrida di file grandi
    • 
                                    

                                

                                

                                    
Cosa NON fare

                                    
    
                                        
  • NON usare Cesare/Vigenère per dati reali (solo didattica)
    • 
                                        
  • NON usare MD5/SHA-1 per sicurezza (solo checksum legacy)
    • 
                                        
  • NON usare password corte o dizionario
    • 
                                        
  • NON condividere mai chiavi private RSA
    • 
                                        
  • NON cifrare file grandi direttamente con RSA
    • 
                                        
  • NON implementare crittografia custom (usa librerie testate)
    •
